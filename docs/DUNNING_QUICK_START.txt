
2. **Enable in code:**

3. **Activate scheduler:**
   - Open `server/index.ts`

## 📚 HOW TO USE

### Generate Dunning Notices (Manual)

1. **Navigate:** `/transactions/dunning-management`
2. **Select Tab:** "Overdue Accounts"
3. **Choose Procedure:** Select "Standard Dunning" (or your procedure)
4. **Click:** "Generate Notices" button

**What happens:**
- System finds all overdue invoices
- Calculates dunning level based on days overdue
- Creates dunning history records
- Sends emails (if enabled)
- Updates customer dunning info
- Blocks customers if > 45 days overdue

### View Dunning History

1. **Navigate:** `/transactions/dunning-management`
2. **Select Tab:** "Reminder History"
3. **Filter:** By status, level, or search customer

### Mark Payment Received

1. **In History Tab:** Find the dunning notice
2. **Click Actions:** → "Mark as Paid"

**What happens:**
- Clears billing_documents (outstanding_amount = 0)
- Clears ar_open_items
- Unblocks customer (dunning_block = false)
- Updates dunning status to 'paid'

### Create Custom Procedure

1. **Navigate:** `/transactions/dunning-management`
2. **Select Tab:** "Procedures"
3. **Click:** "New Procedure"
4. **Configure:**
   - Level 1 days: 7
   - Level 2 days: 14
   - Level 3 days: 21
   - Final Notice: 30
   - Blocking: 45
   - Legal: 60
   - Interest rate: 12% (annual)
   - Dunning fee: $25

## 📊 NEW FEATURES

### Email Templates

4 professional templates included:

- **Level 1:** Friendly reminder (7-13 days)
- **Level 2:** Second reminder (14-29 days)  
- **Level 3:** Final notice (30-59 days)
- **Level 4:** Legal action warning (60+ days)

Templates use placeholders:
- `[CustomerName]` - Customer's name
- `[Amount]` - Dunning amount with interest
- `[DaysOverdue]` - Number of days overdue

### Automatic Customer Blocking

When a customer reaches:
- 45+ days overdue AND
- Dunning Level 3 or higher

System automatically sets `dunning_block = true`

**To enforce in sales orders**, add this check:
```typescript
// In your sales order creation
const customer = await pool.query(
  'SELECT dunning_block, name FROM erp_customers WHERE id = $1',
  [customer_id]
);

if (customer.rows[0]?.dunning_block) {
  throw new Error(`Customer ${customer.rows[0].name} is blocked due to overdue payments`);
}
```

### Run Logging

Every dunning run is logged:
```sql
SELECT * FROM dunning_run_log ORDER BY id DESC;
```

Shows:
- When run occurred
- How many notices generated
- Total amount
- Any errors
- Run type (manual vs scheduled)

## 🎯 TYPICAL WORKFLOW

1. **Daily (if automated):** System runs at 6 AM
2. **Finds Overdue:** Queries billing_documents for overdue invoices
3. **Calculates Level:** Based on days overdue
4. **Generates Notices:** Creates dunning_history records
5. **Sends Emails:** Uses templates to email customers
6. **Updates Customers:** Sets last_dunning_date, last_dunning_level
7. **Blocks Severe Cases:** Sets dunning_block=true for 45+ days
8. **Logs Run:** Records in dunning_run_log

## ⚙️ CUSTOMIZATION

### Change Email Templates

1. **Database Update:**
   ```sql
   UPDATE dunning_email_templates
   SET subject = 'Your custom subject',
       body_html = '<p>Your custom HTML</p>'
   WHERE dunning_level = 1;
   ```

2. **Use Placeholders:**
   - `[CustomerName]`
   - `[Amount]`
   - `[DaysOverdue]`
   - `[InvoiceNumbers]`

### Change Run Schedule

In `dunning-scheduler.ts`, change cron expression:
```typescript
cron.schedule('0 6 * * *', ...);  // Daily at 6 AM
cron.schedule('0 6 * * 1', ...);  // Every Monday at 6 AM  
cron.schedule('0 */6 * * *', ...); // Every 6 hours
```

### Change Blocking Threshold

In procedure configuration:
- Default: 45 days
- Adjust `blocking_days` in procedure settings
- System automatically blocks when threshold reached

## 🆘 TROUBLESHOOTING

### "No overdue accounts found"

**Check:**
```sql
SELECT COUNT(*) FROM billing_documents
WHERE outstanding_amount > 0
AND due_date < CURRENT_DATE
AND posting_status IN ('POSTED', 'OPEN');
```

If zero, you need overdue invoices first.

**Solution:** Run `node scripts/insert-dunning-test-data.js`

### Emails not sending

**Check console logs:** Should see `📧 Dunning Email` lines

**If says "actual sending disabled":**
1. Add SENDGRID_API_KEY to .env
2. Uncomment SendGrid code in dunning-email.ts
3. Restart server

### Customer not blocked after 45 days

**Check:**
1. Dunning level must be >= 3
2. Days overdue must be >= `blocking_days` in procedure
3. Look for console log: `🚫 CustomerName: BLOCKED`
