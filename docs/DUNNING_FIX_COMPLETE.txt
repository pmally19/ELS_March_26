### 4. CUSTOMER BLOCKING ENFORCEMENT ✅ IMPLEMENTED

**In Scheduler Service (`dunning-scheduler.ts` lines 144-152):**

```typescript
if (daysOverdue >= procedure.blocking_days && dunningLevel >= 3) {
  await pool.query(
    `UPDATE erp_customers SET dunning_block = true WHERE id = $1`,
    [account.customer_id]
  );
  console.log(`🚫 ${account.customer_name}: BLOCKED`);
}
```

**Behavior:**
- Customers automatically blocked when:
  - Days overdue >= blocking_days threshold (default: 45 days)
  - Dunning level >= 3 (Final Notice or higher)
- Block persists until payment received
- When marking paid: automatically unblocks customer

**To Enforce in Sales Orders:**
Add to `server/routes/sales-orders.ts`:
```typescript
const customer = await pool.query(
  'SELECT dunning_block FROM erp_customers WHERE id = $1',
  [customer_id]
);
if (customer.rows[0]?.dunning_block) {
  throw new Error('Customer blocked - payment overdue');
}
```

## ENABLING AUTOMATED DAILY RUNS

### Option 1: Node-Cron (Recommended)

1. Install dependency:
   ```bash
   npm install node-cron @types/node-cron
   ```

2. In `server/services/dunning-scheduler.ts`, uncomment lines 174-185:
   ```typescript
   import cron from 'node-cron';
   
   cron.schedule('0 6 * * *', async () => {
     console.log('\n⏰ Scheduled Dunning Run Starting...');
     try {
       await executeDunningRun('scheduled', {
         sendEmails: true,
         sendLetters: false,
       });
     } catch (error) {
       console.error('Scheduled dunning run failed:', error);
     }
   });
   ```

3. Call `scheduleDunningRuns()` in `server/index.ts`:
   ```typescript
   import { scheduleDunningRuns } from './services/dunning-scheduler';
   scheduleDunningRuns(); // Add after app.listen()
   ```

4. Restart server - dunning now runs daily at 6:00 AM automatically

### Option 2: External Cron Job

Set up OS-level cron to call API:
```bash
0 6 * * * curl -X POST http://localhost:5001/api/dunning/run-scheduled \
  -H "Content-Type: application/json" \
  -d '{"send_emails": true}'
```

## SUPPORT & TROUBLESHOOTING
================================================================================

### Common Issues

**Q: Emails not sending?**
A: Check console - should see "📧 Dunning Email" logs
   If says "actual sending disabled", need to:
   1. Add SENDGRID_API_KEY to .env
   2. Uncomment SendGrid code in dunning-email.ts

**Q: Customers not getting blocked?**
A: Check:
   1. Days overdue >= blocking_days in procedure
   2. Dunning level >= 3
   3. Check customer.dunning_block field in database

**Q: Duplicate notices being sent?**
A: Scheduler prevents duplicates within 7 days
   Check customer.last_dunning_date and last_dunning_level

**Q: Payment not clearing invoices?**
A: Verify:
   1. Using PUT /api/dunning/history/:id endpoint
   2. Setting payment_received = true
   3. Check console logs for "✓ Cleared billing documents"
